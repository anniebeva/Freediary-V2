# Правила разработки проекта Freediary

## Backend правила
- Для бэкенда используется FastAPI
- В командной строке Windows и Powershell не использовать оператор `&&`
- Использовать отдельные команды для каждой операции

## Общие принципы разработки

1. **Тестирование изменений**
   - Перед внесением любых изменений необходимо тщательно протестировать код
   - Проверять работоспособность функционала после каждого изменения
   - Убеждаться, что новый код не нарушает существующую функциональность
   - Проводить ручное тестирование всех сценариев использования

2. **Процесс внесения изменений**
   - Всегда проверять влияние изменений на смежные компоненты
   - Использовать консоль разработчика для отладки
   - Проверять корректность типизации
   - Тестировать предельные и краевые случаи

3. **Критерии качества кода**
   - Читаемость и понятность кода
   - Минимальная связанность компонентов
   - Соответствие принципам SOLID
   - Наличие комментариев для сложных участков

4. **Валидация и обработка ошибок**
   - Всегда предусматривать обработку возможных ошибок
   - Добавлять информативные сообщения об ошибках
   - Использовать механизмы превентивной валидации

5. **Производительность**
   - Минимизировать количество рендеров
   - Использовать мемоизацию где необходимо
   - Оптимизировать сложные вычисления

## Специфические правила для Freediary

- Все формы должны иметь валидацию
- Использовать иммутабельные обновления состояния
- Придерживаться типизации, определенной в src/types/Training.ts
- Все даты и числовые значения должны проходить строгую проверку

## Правила тестирования

1. **Перед коммитом**
   - Запускать все тесты
   - Проверять работу на разных устройствах и разрешениях
   - Тестировать все сценарии пользовательского взаимодействия

2. **Виды тестирования**
   - Функциональное тестирование
   - Тестирование пользовательского интерфейса
   - Тестирование граничных значений
   - Проверка обработки ошибок

3. **Чек-лист тестирования**
   - [ ] Корректность ввода данных
   - [ ] Валидация форм
   - [ ] Навигация между страницами
   - [ ] Сохранение и редактирование данных
   - [ ] Обработка edge-cases

## Рекомендации по отладке

- Использовать console.log() для отладки
- Применять React DevTools
- Использовать TypeScript для статической проверки типов
- Добавлять информативные комментарии при сложной логике
